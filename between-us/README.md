# Between Us — Android App Bundle (.aab) project

This is a ready-to-build Android project that wraps the **Between Us** web app
(in `www/`) using [Capacitor](https://capacitorjs.com). Building it produces the
`.aab` (Android App Bundle) you upload to the Google Play Console.

Everything is scaffolded and pre-configured. You only need to run the final
compile step on a machine with the Android toolchain — that step can't run in the
chat sandbox because it downloads build components from Google's servers.

---

## What's inside
- `www/` — the web app (HTML/CSS/JS, manifest, icons). Replace with your real app anytime.
- `android/` — the native Android project (generated by Capacitor).
- `between-us-upload.keystore` — a **sample** signing key so the project builds immediately.
- `keystore.properties` — points the release build at that key.
- `package.json`, `capacitor.config.json` — Capacitor config.

App ID: `com.betweenus.app` · Version: `1.0` (versionCode 1)

---

## APK vs AAB
- **APK** = `app-release.apk` — install this directly on an Android phone to run the app.
- **AAB** = `app-release.aab` — upload this to the Google Play Console (you don't install it).

## Build — three ways

### Option A: GitHub Actions (no local Android setup)
1. Push this folder to a GitHub repo.
2. Actions tab → "Build Android (APK + AAB)" → **Run workflow**.
3. Download `between-us-apk` (and `between-us-aab`) from the run's Artifacts.
GitHub's servers can reach Google, so this builds where the chat sandbox can't.

### Option B: Android Studio
1. Install **Android Studio** (it bundles the Android SDK).
2. Open the `android/` folder, let Gradle sync.
3. **Build → Build Bundle(s) / APK(s) → Build APK(s)** (or Build Bundle(s)).
4. Outputs:
   - APK: `android/app/build/outputs/apk/release/app-release.apk`
   - AAB: `android/app/build/outputs/bundle/release/app-release.aab`

### Option C: Command line
Requires a JDK (17+) and the Android SDK, with `ANDROID_HOME` set.
```bash
cd android
./gradlew assembleRelease    # APK
./gradlew bundleRelease       # AAB
# Windows: gradlew.bat ...
```

### Install the APK on a phone
Copy `app-release.apk` to the phone and open it (allow "install from unknown
sources"), or run `adb install app-release.apk`.

## ⚠️ Before you publish — use your OWN key
The included keystore is a sample so the project builds on first try. For a real
Play Store release you **must** create and safely store your own upload key — if
you lose it you can't update the app.

```bash
keytool -genkeypair -v -keystore my-upload.keystore -alias my-alias \
  -keyalg RSA -keysize 2048 -validity 10000
```
Then edit `keystore.properties` to point at your new keystore/alias/passwords
(see `keystore.properties.example`). Keep the keystore file and passwords private.

---

## Updating the app later
1. Edit files in `www/` (or drop in your real web build).
2. `npm install` (if `node_modules` is missing), then `npx cap sync android`.
3. Bump `versionCode` and `versionName` in `android/app/build.gradle`.
4. Rebuild the bundle.
